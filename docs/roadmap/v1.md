## Rules for staying on track (read this when you're tempted to deviate)

1. **No new module starts before the previous sprint is tagged.** If Sprint 3 runs long, Sprint 4 starts late — it does not start in parallel.
2. **No feature from `v2.md` enters a sprint here.** If you think of something good mid-sprint, write it in `backlog.md` under "Parked ideas" and keep moving.
3. **If a day's tasks aren't done, they roll to the next day within the same sprint — they don't expand the sprint.** If a whole sprint is consistently behind, the fix is cutting scope for that module, not extending the calendar indefinitely.
4. **Sunday is off.** Not a catch-up day. This is what keeps 9 weeks realistic instead of becoming 9 weeks of burnout.
