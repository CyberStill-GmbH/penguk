# Glossary — Penguk

## Problem

A competitive programming problem that has been solved or is scheduled for review.

---

## Platform

An external competitive programming service integrated with Penguk (e.g., LeetCode, Codeforces).

---

## Review

A scheduled repetition session for a solved problem.

---

## Review Queue

The collection of problems scheduled to be reviewed on a specific day.

---

## Spaced Repetition

A learning technique that schedules reviews at increasing intervals to improve long-term retention.

---

## Pattern

An algorithmic category used to classify problems.

Examples:

- Dynamic Programming
- Graphs
- Trees
- Greedy
- Prefix Sum
- Binary Search

---

## Integration

A connection between Penguk and an external platform used to synchronize user data.

---

## Synchronization

The process of importing or updating information from an external platform.

---

## Repository

A GitHub repository connected to Penguk for reading Markdown notes.

---

## Note

A Markdown document associated with one or more solved problems.

---

## Upsolve

The process of solving a contest problem after the contest has ended.

---

## Contest

A competitive programming competition held by an external platform.

---

## Tag

A label assigned to a problem describing its algorithmic topics.

---

## Difficulty

The estimated complexity level of a problem.

---

## Rating

A numerical value representing either:

- the difficulty of a problem, or
- a user's competitive programming rating on a supported platform.

---

## Dashboard

The main interface displaying learning progress, statistics, and scheduled reviews.

---

## User

An authenticated person using Penguk.

---

## Goal

A measurable learning objective defined by the user.

---

## Review History

The chronological record of every review performed for a problem.
